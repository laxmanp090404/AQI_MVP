# AQI Web Service

## Overview
The AQI Web service is a React + Vite web application for visualizing and interacting with AQI predictions generated by the backend AI service.

It provides three major experiences:
- Dashboard for city-wise prediction and KPI cards
- Map View with heatmap, markers, and CPCB legend
- Extensive Report with multi-chart analysis and compact feature table

The UI consumes REST APIs exposed by the backend (`aqi-ai-service`) and renders prediction outputs such as AQI, PM2.5, category, and prediction dates.

## Installation and Setup

### Prerequisites
- Node.js 18+
- npm 9+
- Backend service running at `http://localhost:5000` (default)

### Environment Variables
Create or update `.env`:

```env
VITE_API_URL=http://localhost:5000
```

### Install Dependencies
```bash
npm install
```

### Run Development Server
```bash
npm run dev
```

### Build for Production
```bash
npm run build
```

### Preview Production Build
```bash
npm run preview
```

## Architecture

### High-level flow
Browser UI -> React Pages/Components -> `src/services/api.js` -> Flask Backend APIs -> MongoDB-stored predictions

### Frontend architecture style
- Component-driven React app
- Route-based page composition with `react-router-dom`
- REST integration via Axios
- Visualization with Recharts and Leaflet
- Styling with Tailwind CSS utility classes

### Routing
- `/` -> Dashboard
- `/map` -> Map view
- `/extensive-report` -> Analytical report view

## Functions of Each File (Extensive)

### Root and Tooling Files
- `package.json`
	- Declares scripts (`dev`, `build`, `preview`, `lint`) and dependencies.
- `.env`
	- Supplies `VITE_API_URL` used by Axios client.
- `vite.config.js`
	- Configures Vite with React plugin.
- `postcss.config.js`, `tailwind.config.js`
	- Configure Tailwind CSS pipeline.
- `index.html`
	- Root HTML shell containing the mount node (`#root`).

### Application Bootstrap
- `src/main.jsx`
	- Creates React root and mounts `<App />` inside `StrictMode`.
- `src/index.css`
	- Includes Tailwind directives (`@tailwind base/components/utilities`).
- `src/App.css`
	- Currently empty, reserved for additional global styles.

### App Shell and Routing
- `src/App.jsx`
	- Defines sticky top navigation.
	- Uses `NavLink` to style active/inactive route pills.
	- Registers route-to-page mapping for Dashboard, Map View, and Extensive Report.

### API Layer
- `src/services/api.js`
	- `predictAQI(city)`: POST request to trigger prediction for selected city.
	- `getPredictions()`: GET all predictions for map rendering.
	- `getHistory(city)`: GET city-specific prediction timeline.
	- `getLatestPredictions()`: GET latest prediction per city (grouped in backend).
	- `getExtensiveReport()`: GET wide feature report payload for analytics page.

### Shared Utility
- `src/utils/cities.js`
	- Exports static city-to-coordinate map used by Leaflet components and dropdowns.

### Pages
- `src/pages/Dashboard.jsx`
	- Handles city selection and prediction trigger.
	- Loads latest cards and selected-city AQI history.
	- Computes KPI summaries (city count, average AQI, category counts).
	- Sorts cards by prediction date (latest first).
	- Renders prediction form, current prediction card, latest cards, and trend chart.

- `src/pages/MapPage.jsx`
	- Fetches prediction data for geospatial visualization.
	- Configures Leaflet map (`MapContainer`, `TileLayer`).
	- Composes heatmap, city markers, and CPCB legend overlays.

- `src/pages/ExtensiveReport.jsx`
	- Loads extensive report and latest predictions in parallel.
	- Merges data by city to guarantee PM2.5/AQI visibility in charts and table.
	- Computes summary stats (average AQI, best/worst city).
	- Implements CPCB AQI color mapping and category labels.
	- Provides multi-chart analytics:
		- City AQI comparison
		- PM2.5 signal mix by city (dual axis)
		- Selected-city lag trend
		- Humidity vs AQI scatter
	- Adds horizontal scrolling for city-dense charts.
	- Displays compact tabular report with core metrics.

### Components
- `src/components/CitySelecter.jsx`
	- Reusable city dropdown bound to selected city state.

- `src/components/PredictionCard.jsx`
	- Displays city-level AQI card with color-coded background, category, PM2.5, and date.

- `src/components/Loader.jsx`
	- Spinner used during API operations.

- `src/components/AQITrendChart.jsx`
	- Responsive Recharts line chart for AQI history over date.

- `src/components/AQIHeatMap.jsx`
	- Builds Leaflet heat layer from latest AQI points by city.
	- Normalizes heat intensity as `aqi / 200`.

- `src/components/AQIMarkers.jsx`
	- Renders city circle markers and detailed popups.
	- Uses prediction-provided color/category labels.

- `src/components/AQILegend.jsx`
	- Shows CPCB color legend overlay on map.

## Features of AQI Frontend
- Modern top-nav app shell with active route states
- Dashboard KPIs and city-wise prediction workflow
- Latest city card grid sorted by prediction recency
- Historical AQI trend chart for selected city
- Interactive India map with heatmap + markers + legend
- Extensive report with chart-first analytics and compact table
- Horizontal scrolling support for charts when many cities are present
- Consistent CPCB AQI color semantics across report visualizations

## API Dependency Contract
Frontend expects backend base URL from `VITE_API_URL` and consumes:
- `POST /predict`
- `GET /predictions`
- `GET /history/<city>`
- `GET /latest-predictions`
- `GET /extensive-report`
